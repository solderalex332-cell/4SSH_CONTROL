# Полная схема PoC

Есть четыре компонента в системе:

1.	Admin-1 (обычный администратор) сидит на своём рабочем хосте (management host, ноутбук/сервер админа).
Подключается по SSH к бастиону (думает, что напрямую к целевому серверу), вводит команды, которые хочет выполнить на объекте управления
2.	Бастион (MITM-прокси / контрольная точка)  промежуточный сервер, на котором запущен код (paramiko-based proxy или ssh-mitm с доработками), который принимает подключение от Admin-1 и сам подключается по SSH к объекту управления (target), перехватывает каждую введённую команду от Admin-1, показывает её Admin-2 в реальном времени, ждёт подтверждения (y/n), если y, то передаёт команду на объект управления, а если n, то блокирует, отправляет Admin-1 сообщение "Command blocked by security policy" и аналогично может перехватывать и подтверждать вывод от объекта
3.	Admin-2 (контролёр / второй администратор / gatekeeper) может сидеть на той же машине (бастионе), на отдельном терминале, или удалённо (через файл, сокет, в PoC проще всего в консоли бастиона), который видит каждую команду в формате:
    ```text
    [ADMIN-2] Command detected: rm -rf /var/log/*
    y = allow, n = block:
    вводит y, Enter (пропустить) или n, Enter (заблокировать)
    ```
4.	Объект управления (target host / управляемый хост / целевой сервер) -реальный сервер, который нужно администрировать (production/dev сервер, роутер, база данных и т.д.), который находится в защищённой сети. Admin-1 не имеет прямого доступа к нему, только через бастион, выполняет одобренные команды и отдаёт вывод обратно через бастион с  Admin-1
